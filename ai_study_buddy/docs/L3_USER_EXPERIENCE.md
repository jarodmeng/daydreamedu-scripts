# AI Study Buddy — User Experience

> Status: **Exploratory** — options and recommendations, not final decisions.
>
> Sources: ChatGPT 5.2 Thinking conversation (3 Mar 2026), supplemented with independent analysis (Opus 4.6 Max, 4 Mar 2026).

---

## Platform

**Recommended: Progressive Web App (PWA)** — a role-based web app that works on iPad, laptop, and phone without maintaining native apps. Benefits:
- Easy login + permissions (parent vs kid)
- File upload / camera scan
- Cheap to host (static frontend + serverless backend)
- Installable on home screen for an "app-like" feel

---

## Three User Interfaces

The app has three "faces," accessed via a profile switch:

1. **Kid UI** — Winston, Emma, Abigail
2. **Parent UI** — dashboard + controls
3. **Admin / Ingestion Review UI** — PDF review + correction

---

## 1. Kid UI: Quest-First (Not Chat-First)

### Design principle

Kids should **not** land on a blank chat box. They should land on a **Quest Board** — a game-like interface showing what to do today, generated by the planner.

### Kid Home (Quest Board)

| Element | Description |
|---------|-------------|
| **Today's Quests** | 2–4 items, each 10–20 min: "Learn quest" (new skill), "Review quest" (spaced repetition), "Boss fight" (weekly timed set) |
| **Streak + XP + Level** | Visible at all times; motivational |
| **Next milestone** | e.g., "WA in 12 days", "PSLE Oral in 23 weeks" |
| **"I'm stuck" button** | Jumps into tutor with the exact quest context |

### Tutor Screen (the "coach")

A chat-style interface **anchored to a specific question object**:
- Shows the question (image crop or text)
- "Show hint" ladder (Hint 1 → Hint 2 → Example → Partial solution)
- "Explain your thinking" box (teach-back)
- "Check my answer" (allowed/limited based on child settings)

### Quick Scan / Upload

A prominent "Scan worksheet" button:
- Camera capture → auto-crop → choose question number (or auto-detect)
- Drops the child straight into the tutor for that question

### Mistakes & Power-ups

A "My Mistakes" page:
- Top 3 mistakes this week
- "Fix-it quests" generated from repeated error tags
- Badges like "Fixed a repeat mistake"

### Grade-Appropriate Modes

| Child | Grade | Mode |
|-------|-------|------|
| **Abigail** | P2 | More voice, shorter quests, more visuals, "guided only" (no answer reveals) |
| **Emma** | P4 | Guided + limited reveal, lots of micro-quizzes |
| **Winston** | P6 | Exam mode: timed sets, method-mark rubrics, strategy feedback |

## 2. Parent UI: Insight + Control (Not Micromanagement)

### Parent Dashboard

| Section | Content |
|---------|---------|
| **This week at a glance** | Time spent per child, streak health, skills improving / declining |
| **Upcoming assessments** | From school calendar |
| **Risk flags** | "Careless error spike", "avoidance (abandoned sessions)", "weak skill not revisited in 14 days" |

### Planner Controls

- Set time budgets (weekday / weekend)
- Choose focus priorities (e.g., "Winston: Math word problems; English editing")
- Set constraints ("no final answers unless teach-back done")

### Weekly Summary Generator

A one-page auto-generated report:
- What changed in mastery
- What to do next week
- What the agent recommends reinforcing offline

---

## 3. Ingestion / Review UI: The Money-Saving Screen

### Paper Review Screen

After a PDF is ingested:
- Page thumbnails
- Auto-detected questions with bounding boxes overlaid
- Extracted score boxes (e.g., "2/4")
- Quick-fix controls for:
  - Question boundaries
  - Marks
  - Question numbering
  - Subject/grade/date tags

Corrections feed back into **template profiles** so future papers from the same school format require less AI intervention.

## How UI Maps to Architecture

| UI element | Backend service |
|------------|----------------|
| Quest Board | Planner Service + Student Model |
| Tutor | Tutor Service + Retriever (question crops + past mistakes + textbook snippets) |
| Scan/Upload | Content Service + Ingestion operators (AI only when needed) |
| Mistakes view | Attempts + Error tags |
| Parent dashboard | Analytics service over event logs |
| Review UI | Ingestion pipeline outputs + edit tools |

## MVP UI (Build First)

1. Kid Quest Board
2. Tutor screen (with hint ladder + teach-back)
3. Upload/Scan
4. Parent weekly summary page
5. Ingestion review screen (basic bounding box + marks correction)

Everything else (leaderboards, avatars, deep analytics) layers later.

> [!NOTE]
> **Opus 4.6 Max analysis** — UX considerations beyond the ChatGPT conversation.

### Multi-modal input is critical for young children

The conversation mentions voice for Abigail (P2) but underestimates how important this is. At age 7–8, typing is slow and frustrating. Consider:

- **Voice-first interaction for Abigail:** Speech-to-text for input, text-to-speech for the tutor's responses. This makes the experience feel more like talking to a real tutor.
- **Photo-first for worksheets:** All three children should be able to take a photo of a question and get help immediately. The "Scan worksheet" button should be the most prominent action.
- **Handwriting input (stretch goal):** Especially for math, allowing children to write their workings on a tablet screen (e.g., iPad with Apple Pencil) could be very powerful for the teach-back step.

### Session length design

Children's attention spans vary significantly by age. Suggested defaults:

| Child | Age | Suggested session | Quest length |
|-------|-----|-------------------|-------------|
| Abigail | ~7–8 | 15–20 min max | 5–8 min per quest |
| Emma | ~9–10 | 20–30 min | 8–12 min per quest |
| Winston | ~11–12 | 30–45 min | 10–20 min per quest |

The UI should **celebrate completion** (not just correctness) and make it easy to stop at a natural break point.

### Offline considerations

If the children use the app on an iPad at home, WiFi reliability matters. Consider:
- **Caching today's quests** so they work offline
- **Queuing attempt logs** to sync when back online
- **Pre-loading question images** for the day's quests

This is a PWA strength — service workers can handle caching and background sync.

### Accessibility

- **Font sizes and contrast** should be larger than typical web apps (target audience is children)
- **Reading level of UI text** should match each child's grade level
- **Error messages** should be encouraging, not punitive ("Let's try that again!" not "Incorrect")

### Open Questions

1. **Which devices will the kids use most?** iPad, laptop, or phone? This drives layout decisions (landscape vs portrait), input method priorities, and session length defaults.
2. **Should the app have a "parent override" mode?** e.g., parent can sit with the child and unlock additional features during supervised study.
3. **Notification strategy:** Push notifications for daily quest reminders? Or is that too intrusive?
